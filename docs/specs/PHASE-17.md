# Phase 17 — Workflows

Implement persistent workflow definitions, versions, nodes, edges, triggers, conditions, branches, schedules, approvals, retries, runs, and step history.

Provide a visual editor and a natural-language builder that produces an editable definition. Execution must use validated deterministic steps with explicit permissions, inputs, retries, and approval gates.

Support draft, test, active, paused, archived, rollback, manual run, scheduling, webhook triggers, cancellation, resume, and sanitized run logs. Add graph validation, mapping, idempotency, retry, schedule, approval, authorization, worker, and Playwright tests.
