# Phase 06 — Tasks, initiatives, approvals, and collaboration

Create durable entities and APIs for tasks, assignments, comments, mentions, notifications, initiatives, blockers, dependencies, schedules, and approvals. Support priorities, due dates, allowed status transitions, completion, cancellation, and linked source records.

Use a central structured activity writer with allowlisted action codes; never accept arbitrary prompt text for activity fields. Comments use the platform encryption path. Approvals support assigned reviewer, projected cost, expiration, approve, reject, request changes, and audit history.

Build functional project UI for list, board/detail, assignment, comments, initiative progress, blockers, approvals, and notifications. Add transition, permission, concurrency, notification, and Playwright tests.
